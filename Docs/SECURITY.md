# Security Policy

## Supported Versions
Only the latest version of CryPTrus 2.0 is currently supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 2.0.x   | :white_check_mark: |
| < 2.0   | :x:                |

## Reporting a Vulnerability
**Do not open a public GitHub issue for security vulnerabilities.**

If you discover a security vulnerability within CrypTrus 2.0, please report it privately. We want to keep our users safe!

### Reporting Process:
1. Email your findings to **sosnacdavid@gmail.com**.
2. Include a detailed description of the vulnerability.
3. Provide steps to reproduce the issue (PoC).
4. Allow us up to 72 hours to acknowledge the report.

### Disclosure Policy:
We follow a coordinated vulnerability disclosure model. We ask that you do not share the vulnerability publicly until we have had the opportunity to provide a fix for our users.

## Our Security Philosophy
CrypTrus 2.0 is built on a **Zero-Knowledge** architecture. We never store Master Keys, and all encryption happens client-side. Our goal is to provide a "Privacy-First" environment using industry-standard AES-256-GCM.
